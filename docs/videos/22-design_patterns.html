Concepts

  • pattern
    solution applied more than once to a problem when it occurs in a particular context

  • Types of design patterns (for software development)

    • creational pattern
      pattern that helps with the creation of an instance of an object
    • structural pattern
      pattern that helps with the setup of communication pathwayw between different groups of objects
    • behavioral pattern
      pattern that helps with the partitioning of system behaviors into discrete classes

  • What is purpose of software design patterns?

    • Provide strategies for minimizing dependencies during ystem design.

  • command (in software development) ~- an object with the role of storing all the information required for executing a particular action. See https://www.baeldung.com/java-command-pattern.

  • Command pattern
    an interface, typically named Command, with a single method, “Execute”.

  • a command is an object, and objects can hold state; so a comman object that performs a function could save information about what it did (i.e., it remembers the relevant details about its actions); this means it can undo those actions at a later time
